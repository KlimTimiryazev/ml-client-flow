## Задача

Бинарная классификация: предсказать, уйдёт ли клиент (churn = Yes/No) по поведенческим и платёжным признакам подписки.

Практическое применение — заблаговременное выявление клиентов с высоким риском оттока для удержания.

---

## Датасет

`customer_subscription_churn_usage_patterns.csv` — 1000 записей о клиентах подписочного сервиса.

| Признак | Тип | Описание |
|---|---|---|
| `user_id` | int | Идентификатор (удаляется перед обучением) |
| `signup_date` | date | Дата регистрации |
| `plan_type` | категориальный | Тип плана: Basic / Premium |
| `monthly_fee` | float | Ежемесячная плата (199 / 699 ₽) |
| `avg_weekly_usage_hours` | float | Среднее время использования в неделю |
| `support_tickets` | int | Число обращений в поддержку |
| `payment_failures` | int | Число неуспешных платежей (0–5) |
| `tenure_months` | int | Срок подписки в месяцах |
| `last_login_days_ago` | int | Дней с последнего входа |
| `churn` | Yes / No | **Целевая переменная** |

**Баланс классов:** ~57% No / ~43% Yes — дополнительная балансировка не требуется.

## EDA — ключевые выводы

### Мультиколлинеарность
`plan_type` и `monthly_fee` полностью коллинеарны (Basic = 199₽, Premium = 699₽).  
**Решение:** удалить `plan_type` перед обучением — оставить только `monthly_fee`.

### Наиболее информативные признаки (по связи с оттоком)
| Признак | Наблюдение |
|---|---|
| `last_login_days_ago` | Ушедшие клиенты давно не заходили — сильный поведенческий сигнал |
| `support_tickets` | Высокое число обращений коррелирует с оттоком |
| `payment_failures` | Проблемы с оплатой — явный предиктор |

### Распределения
Все числовые признаки распределены равномерно, аномальных значений не обнаружено.

### Мультиколлинеарность между остальными признаками
Отсутствует (confirmed по тепловой карте корреляций).
